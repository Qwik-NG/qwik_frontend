# Frontend Implementation Rules

## Copilot Rules

- Work phase-by-phase; do not combine phases.
- Never redesign approved UI.
- Never touch unrelated files.
- Always run npm run build before completion.
- Always check git status before and after changes.
- Prefer reusable components over duplicated UI.
- Preserve teammate work and avoid large unrelated diffs.
- Keep frontend ownership separate from backend ownership.
- Do not implement backend functionality or API logic.
